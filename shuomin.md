# RPKI 冲突检查器 - 超级简单说明书

## 🎯 这个项目是做什么的？

想象一下，全世界有很多很多的"身份证"，叫做 **RPKI 证书**。

这些证书就像是互联网上的"身份证"，用来证明："我是谁，我可以管理哪些网络地址"。

但是有个问题：**有时候，两张不同的"身份证"会说同样的话**，这就产生了"冲突"！

**我们的项目就是一个侦探**，它的工作是：
- 🔍 收集所有这些"身份证"
- 📖 读取他们的信息
- ⚠️ 发现哪些"身份证"发生了冲突
- 💾 把这些信息记录下来

---

## 🏗️ 项目的各个部分是什么？

### 1️⃣ **下载器** (好比是邮递员)
- 从互联网上很远的地方下载 RPKI 证书
- 就像邮递员去邮局取信一样

### 2️⃣ **读取器** (好比是信件打开机)
- 打开每一张"身份证"
- 读出里面写了什么信息
- 就像把信封打开，读出信里的内容

### 3️⃣ **检查器** (好比是质检员)
- 比较不同的"身份证"
- 看看有没有两张证书说同样的话
- 如果有重复，就标记为"冲突"

### 4️⃣ **记录员** (好比是档案馆)
- 把所有的信息保存在一个大柜子里（数据库）
- 这样以后需要时，就能快速查找

### 5️⃣ **区块链记账员** (好比是公证处)
- 把重要的记录再写到一个特殊的账本里
- 这个账本就是区块链，特别安全，不能改动

## 🚀 怎么用这个项目？

### 第一步：准备环境
```
就像做饭前要准备好厨房一样，我们需要：
- 安装 Java（计算机能"听懂"Java）
- 安装 MySQL（准备好大柜子）
- 安装 Docker（一个虚拟机管家）
```

### 第二步：启动数据库
```bash
docker start rpki-mysql
```
这就像打开冰箱一样，让数据库开始工作。

### 第三步：启动应用程序
```bash
java -jar target/conflict-checker-backend-1.0.0.jar
```
这就像启动一台机器一样，让我们的侦探开始工作。

### 第四步：让侦探开始工作
打开浏览器，访问：
```
http://localhost:8081/api/cert/auto-process
```

或者在命令行输入：
```bash
curl -X POST http://localhost:8081/api/cert/auto-process
```

这就像按下"开始"按钮，侦探就开始收集、读取和检查证书了。

## 📊 项目运行后会做什么？

### 1. 下载阶段 ⬇️
侦探从网络上下载最新的 RPKI 证书（可能是几千张）。
就像你妈妈给你列出全班同学的名单一样。

### 2. 读取阶段 📖
侦探打开每一张证书，读出信息：
- 谁发行的这张证书
- 什么时候失效
- 能管理哪些网络地址
- 能管理哪些 ASN（就像网络的"序列号"）

### 3. 检查阶段 🔍
侦探比较所有证书，找出冲突：
- 有没有两个地址段重叠？
- 有没有两张证书说同一个 ASN 属于他们？
- 如果有，就标记为"冲突"

### 4. 保存阶段 💾
侦探把所有发现存放到数据库中，这样就可以：
- 快速查找某个证书的信息
- 看有多少张证书有冲突
- 看哪些证书已经上链了

### 5. 区块链记账 📝
最后，这些记录也被写到特殊的账本（区块链）上，确保永远不会被改动。

## 🔍 如何查看处理结果？

### 查看统计信息
```
http://localhost:8081/api/cert/stats
```
你会看到：
- 总共有多少张证书
- 有多少张有冲突

### 查看所有冲突证书
```
http://localhost:8081/api/cert/conflicts
```
你会看到所有"身份证"打架的情况。

### 查看已上链的证书
```
http://localhost:8081/api/cert/on-fabric
```
你会看到已经写进区块链账本的记录。

## 🔧 出现问题怎么办？

### 问题 1：显示"数据库连接失败"
**原因**：冰箱还没打开
**解决办法**：
```bash
docker start rpki-mysql
```

### 问题 2：显示"端口被占用"
**原因**：可能有另一个程序在用 8081 端口
**解决办法**：关掉其他占用这个端口的程序，或者改用其他端口

### 问题 3：显示"证书下载失败"
**原因**：网络可能不好，或者网站暂时关闭了
**解决办法**：等一会儿再试

### 问题 4：显示"编译失败"
**原因**：Java 或依赖包没装好
**解决办法**：
```bash
mvn clean install
```
